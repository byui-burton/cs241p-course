13 Teach : Team Activity

Overview

This activity will help you practice writing shell scripts in Linux.

Your assignment is to write a shell script that accepts a directory as a command line parameter, and then iterates through the .py files in that directory.

Instructions

Core Requirements

  1. Create a shell script that accepts a file as a command line parameter and then displays it to the screen as follows: (The dollar sign below just means you are typing it at the command line)

    Make your script executable and ensure that it runs.

    2. 
$ ./lineCounter.sh myFile.py
Running on file: myFile.py

  2. Have your script count the number of lines in that file and display it to the screen. For example:

    3. 
$ ./lineCounter.sh myFile.py 
Running on file: myFile.py
127 myFile.py

  3. Change your script to accept a directory (rather than a file) as the command line parameter. Then have it iterate through all .cpp files in that directory and for each one, display the name of the file and the number of lines in that file.

    4. 
$ ./lineCounter.sh /home/cs241
Running on directory: /home/cs241
Files in the directory:
40 /home/cs241/check03b.py
63 /home/cs241/check04b.py
62 /home/cs241/check05a.py
37 /home/cs241/check07a.py
55 /home/cs241/check07b.py

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Change your script so that the number of lines per file is saved to an output file, rather than to the screen. Specify the output file with another parameter (e.g., ./lineCounter.sh /home/cs241 pyLines.txt)

  2. Change your script so that it looks through sub-directories (just one level deep) for .py files as well.

  3. Change your script so that in additional to accepting a directory name, it also accepts a list of potential extensions (e.g., ./lineCounter.sh /home/cs241 pyLines.txt py cpp txt). Then, have it count the lines in all of those types of files.
